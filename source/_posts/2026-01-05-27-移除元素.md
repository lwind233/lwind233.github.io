遍历一遍数组，一遍遍历一遍更新元素，同时维护一个偏移量，每有一个需要被移出的元素，偏移量就加一，最后偏移量就是被移除的元素数量

```java
class Solution {
    public int removeElement(int[] nums, int val) {
        int offset=0;
        int l = nums.length;


        for(int i=0;i+offset<=l;i++){
            while(i+offset<=l-1 && nums[i+offset]==val){
                offset++;
            }
            if(i+offset>=l){
                break;
            }
            nums[i] = nums[i+offset];
        }
        return l-offset;
    }
}
```

不过我的思路复杂了，还有更简单的实现方法

```java
class Solution {
    public int removeElement(int[] nums, int val) {
        int offset=0;
        int l = nums.length;


        for(int i=0;i<=l-1;i++){
            if(nums[i]!=val){
                nums[offset] = nums[i];
                offset++;
            }
        }
        return offset;
    }
}
```

本身整个代码也是双指针的思想
