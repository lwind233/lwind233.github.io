原地哈希，将提供的数组作为哈希表

注意到，当数组长度为 n 时，答案一定落在 [1, n+1]。如果 1..n 都出现，则答案为 n+1，因此 >n 的正数对答案无贡献。

先遍历第一遍数组，将所有的0替换为-1（0对我们没有意义，将所有的0去掉，以将0作为标记）

第二次遍历数组，假如i属于[1,n]，那么将nums[i-1]置为0，若i-1位置上的数字也是[1,n]之中，重复这个步骤

（最佳的实现是将所有的小于等于0的数替换为n+1，然后若i属于[1,n]，只需将将nums[i-1]置为相反数即可，这样就不必用复杂的while链式调整）

第三次遍历数组，找出第一个不是0的数字。（负号标记法只需要找出第一个负数即可）

（while链式替换解法）

```java
class Solution {
    public int firstMissingPositive(int[] nums) {
        int len = nums.length;
        for(int i=0;i<=len-1;i++){
            if(nums[i]==0) nums[i] = -1;
        }

        for(int i=0;i<=len-1;i++){
            if(nums[i]>=1 && nums[i]<=len){
                int idx = nums[i]-1;
                while(nums[idx]>=1 && nums[idx]<=len){
                    int tmp = nums[idx];
                    nums[idx] = 0;
                    idx = tmp-1;
                }
                nums[idx]=0;
            }
        }

        int fIdx=0;
        for(;fIdx<=len-1;fIdx++){
            if(nums[fIdx]!=0) return fIdx+1;
        }
        return fIdx+1;
    }
}
```
