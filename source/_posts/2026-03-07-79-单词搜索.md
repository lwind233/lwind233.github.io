维护当前在匹配第idx个位置的字符

如果当前位置idx已经超过了字符串长度，说明之前的所有字符全部匹配成功，即可以搜索到这个单词，返回true

每一层先判断i,j的位置是否合法，然后判断当前位置是否已经被使用过，最后判断当前位置的字符与待匹配字符是否一致，是的话当前字符匹配成功。

随后idx+1，将当前位置标记为已经使用，然后朝四个方向进行遍历。

```java
class Solution {
    boolean result=false;

    public boolean exist(char[][] board, String word) {
        for(int i=0;i<=board.length-1;i++){
            for(int j=0;j<=board[0].length-1;j++){
                traceBack(board,word,i,j,0);
                if(result) return true;
            }
        }
        return false;
    }

    void traceBack(char[][] board, String word, int i, int j, int idx){
        if(result) return;

        if(idx==word.length()){
            result=true;
            return;
        }

        if(i<0 || j<0 || i>board.length-1 || j>board[0].length-1) return;
        if(board[i][j]=='#') return;
        if(board[i][j]!=word.charAt(idx)) return;

        char tmp = board[i][j];
        board[i][j]='#';
        traceBack(board,word,i,j+1,idx+1);
        traceBack(board,word,i,j-1,idx+1);
        traceBack(board,word,i+1,j,idx+1);
        traceBack(board,word,i-1,j,idx+1);
        board[i][j]=tmp;
    }
}
```

### 优化点

* 可以直接修改board用于进行标记，通过回溯撤销，这样可以省略掉一个字符
* 可以先判断board中的各个字符数是否比字符串中的多
* 可以判断哪个字符最少，如果开头字符数大于结尾字符数，那把字符串翻转后可能搜的更快
